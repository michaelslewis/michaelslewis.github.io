Yelp's Search API

While the sample code on the previous page was very basic, returning relatively broad results of 'food' located in 'Corvallis', primarily as a means to demonstrate the setup of OAuth, the Yelp Search API provides a myriad of parameters to allow us to further refine our results. Yelp provides these parameters in the Search API tab of their documentation page.

About Ajax
When we run our search, it is processed asynchronously, using Ajax. Ajax stands for Asynchronous Javascript and XML, but for our purposes, the "Asynchronous JavaScript" part is the most relevant aspect of the name. While the response data can be received in the format of XML, we will be using JSON (more about this on the following page). The Ajax technology is popular because it allows a GET request from the client to be made to the server and processed simultaneously, as the rest of the page loads. One of the main benefits is efficiency. For example, if we run a search, and then change a portion of that and run a subsequent search, that second search does not need to reload all of the information, only the part that was changed. In contrast with searches that are merely synchronous, asynchronous calls can load much faster, which may result in a much better experience for the end user.

In general, aside from setting the 'location' parameter, the 'term' parameter will likely be our most commonly used parameter for these searches. This can be set to general values such as the 'food' search from the previous sample code. But Yelp becomes much more powerful when we refine or combine terms. For example, instead of the term 'food', we could specify 'Chinese food'. In the following sample code, we will conduct a more advanced search, where we constrain our search to only "Chinese Restaurants". We will also use the additional 'limit' parameter to narrow the number of results returned to only '10' restaurants. Furthermore, we will set the 'sort' parameter to a value of '2', which will return our restaurants in order of the highest rated. Lastly, we will use the 'radius_filter' parameter with a value of '8000'. This last parameter measures a distance from the center of the specified location, and returns only those businesses within that radius. Since it is measured in meters, our value of '8000' will limit the results to restaurants within approximately 5 miles of Corvallis.

Our parameter/value pairs are defined within a JavaScript object, using JavaScript Object Notation. Remember to separate each parameter/value pair by a comma. Also, please note that there is neither a comma nor a semicolon after the last pair in the object. Lastly, while this format may appear similar to the JSON format (which we will discuss further on the next page), it is not identical. Unlike the notation we use for an object in the JavaScript language, the format used for JSON is intended for data exchange and independent of language, so it will require certain syntactical differences, including the use and placement of quotation marks.
